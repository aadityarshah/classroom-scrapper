Probabilistic Reasoning: Foundations

1. The Deterministic Baseline & Its Limitations​

Deterministic Reasoning​

  • Core Concept: No randomness. Same input always yields same output.
  • Assumes a "perfect" world model where all conditions are known and fixed.
  • Model: Input: X -$>$ Fixed Rules (Logic) -$>$ Output: Y
  • Examples: Logic gates, standard sorting algorithms.

The Challenge of Reality​

  • Real-world scenarios introduce uncertainty, making deterministic rules fail.
  • Example: Leave(8:00) =$>$ Arrive(8:30) is often false in reality due to unforeseen factors (traffic, flat tire).

The Qualification Problem​

  • To make a logical rule strictly "True" in the real world, endless conditions are required.
  • Leave(8:00) ∧ ¬Traffic ∧ ¬FlatTire ∧ ... =$>$ Arrive(8:30)
  • Issues:
    • Infinite exceptions possible.
    • Impossible to pre-observe all conditions.
  • Pure logic (rule-based systems) becomes intractable due to "too many exceptions."

2. Embracing Uncertainty: Probabilistic Systems​

Shift in Paradigm​

  • From True/False to Degrees of Belief.
  • Probabilistic Model: Input -$>$ Probabilistic Model -$>$ Probabilities of Outcomes
  • Key Characteristics:
    • Acknowledges missing information.
    • Assigns probability based on available data.

Sources of Uncertainty​

  • Why AI systems aren't perfectly logical:
    1. Laziness: Too much work to enumerate all rules/conditions.
    2. Theoretical Ignorance: No perfect mathematical model exists for complex systems.
    3. Practical Ignorance: Cannot deploy infinite sensors to measure all variables.
  • Engineering Takeaway: Probability summarizes what we don't know or didn't measure.

Quantifying Belief​

  • P(A)P(A)P(A) represents the strength of belief that event AAA is true.
  • Values range from 0.00.00.0 (disbelief/impossible) to 1.01.01.0 (certainty/guaranteed).
  • Beliefs are dynamic and update as new evidence arrives.

Rational Agency​

  • A Rational Agent maximizes expected utility.
  • Decision-making involves balancing probabilities of outcomes against their associated utilities (costs/benefits).
  • Rationality = Balancing Probability vs. Utility.

3. Core Concepts in Probability​

Random Variable (RV)​

  • A Random Variable (XXX) is a function, not an algebraic variable.
  • It maps an atomic event (ω\omegaω) from the Sample Space (Ω\OmegaΩ) to a value in a specific domain (DDD).
  • Definition: X:Ω→DX: \Omega \to DX:Ω→D
  • Domains (D):
    • Boolean: ⟨T,F⟩\langle T, F \rangle⟨T,F⟩ (e.g., Cavity: True/False)
    • Discrete: {Sunny,Rainy,Cloudy}\{\text{Sunny}, \text{Rainy}, \text{Cloudy}\}{Sunny,Rainy,Cloudy} (e.g., Weather)

Sample Space (Ω\OmegaΩ) & Atomic Events​

  • Sample Space (Ω\OmegaΩ): The set of all possible "worlds" or outcomes.
  • Atomic Event (ω\omegaω): A single, completely specified world state.
  • Constraints: All atomic events are Mutually Exclusive (only one can occur) and Exhaustive (cover all possibilities).
  • Example (Dental Universe): For Cavity (C) and Toothache (T), there are 4 atomic events:
    • ω1:(C=T∧T=T)\omega_1: (C=T \land T=T)ω1​:(C=T∧T=T)
    • ω2:(C=T∧T=F)\omega_2: (C=T \land T=F)ω2​:(C=T∧T=F)
    • ω3:(C=F∧T=T)\omega_3: (C=F \land T=T)ω3​:(C=F∧T=T)
    • ω4:(C=F∧T=F)\omega_4: (C=F \land T=F)ω4​:(C=F∧T=F)

Kolmogorov's Axioms​

  • Three fundamental rules governing probability:
    1. Non-negativity: 0≤P(ω)≤10 \le P(\omega) \le 10≤P(ω)≤1 for any atomic event ω\omegaω.
    2. Normalization: ∑ω∈ΩP(ω)=1\sum_{\omega \in \Omega} P(\omega) = 1∑ω∈Ω​P(ω)=1. The sum of probabilities of all possible atomic events in the sample space is 1.
    3. Additivity (Inclusion-Exclusion Principle): For any two events AAA and BBB: P(A∨B)=P(A)+P(B)−P(A∧B)P(A \lor B) = P(A) + P(B) - P(A \land B)P(A∨B)=P(A)+P(B)−P(A∧B)

4. Types of Probability​

Prior (Unconditional) Probability​

  • Definition: The baseline belief in a state before any evidence or sensors activate.
  • Represents the historical background rate or prevalence of an event.
  • Notation:
    • Scalar: P(Cavity=true)=0.1P(\text{Cavity}=\text{true}) = 0.1P(Cavity=true)=0.1
    • Vector (Distribution): P(Weather)=(0.7,0.2,0.1)P(\text{Weather}) = (0.7, 0.2, 0.1)P(Weather)=(0.7,0.2,0.1) (for Sunny, Rainy, Cloudy)

Joint Probability Distribution (JPD)​

  • Definition: A complete look-up table specifying the probability of every possible atomic event (every combination of variable values).
  • Each row represents an atomic event with its probability.
  • Power: Theoretically, an AI holding the JPD can answer any probabilistic query by summing relevant rows.
  • Anatomy Example (Cavity and Toothache):
    Toothache (T)¬Toothache (¬T)
    Cavity (C)0.120.08
    ¬Cavity (¬C)0.040.76
  • Operations:
    • Verify Axiom 2: 0.12+0.08+0.04+0.76=1.00.12 + 0.08 + 0.04 + 0.76 = 1.00.12+0.08+0.04+0.76=1.0
    • Extract Prior P(C)P(C)P(C): P(C)=P(C∧T)+P(C∧¬T)=0.12+0.08=0.20P(C) = P(C \land T) + P(C \land \neg T) = 0.12 + 0.08 = 0.20P(C)=P(C∧T)+P(C∧¬T)=0.12+0.08=0.20

The Complexity Crisis of JPDs​

  • JPD assumes every variable affects every other variable.
  • For nnn Boolean variables, the JPD requires 2n2^n2n rows/entries.
  • Scalability Issue:
    • n=2n=2n=2 (Dentist): 4 entries
    • n=20n=20n=20: ≈1\approx 1≈1 Million entries
    • n=100n=100n=100 (Autonomous Vehicle): 1.26×10301.26 \times 10^{30}1.26×1030 entries
  • Engineering Reality: JPDs are a catastrophic failure at scale; they cannot be stored or computed for complex real-world problems.

Conditional (Posterior) Probability​

  • Concept: Belief after observing evidence. The probability of event AAA given that event BBB has occurred.
  • Formula: P(A∣B)=P(A∧B)P(B)P(A | B) = \frac{P(A \land B)}{P(B)}P(A∣B)=P(B)P(A∧B)​
  • The sample space is effectively reduced to only include states where BBB is true.
  • Example (Dental): If P(C∧T)=0.12P(C \land T) = 0.12P(C∧T)=0.12 and P(T)=0.16P(T) = 0.16P(T)=0.16, then P(C∣T)=0.120.16=0.75P(C | T) = \frac{0.12}{0.16} = 0.75P(C∣T)=0.160.12​=0.75.

The Product Rule​

  • Rearrangement of the conditional probability formula.
  • Allows calculation of joint probabilities from smaller, local rules.
  • Formulas: P(A∧B)=P(A∣B)P(B)P(A \land B) = P(A | B)P(B)P(A∧B)=P(A∣B)P(B) P(A∧B)=P(B∣A)P(A)P(A \land B) = P(B | A)P(A)P(A∧B)=P(B∣A)P(A)
  • Engineering Value: Enables building complex joint probabilities from prior and conditional probabilities without a full JPD.

5. Bayes' Theorem: The AI Engine​

Inverting Probabilities​

  • Equating the two forms of the Product Rule (P(A∣B)P(B)=P(B∣A)P(A)P(A|B)P(B) = P(B|A)P(A)P(A∣B)P(B)=P(B∣A)P(A)) allows us to "invert" conditional probabilities.
  • This is crucial for diagnostic reasoning (inferring cause from effect).

Bayes' Rule​

  • Formula: P(A∣B)=P(B∣A)P(A)P(B)P(A | B) = \frac{P(B | A)P(A)}{P(B)}P(A∣B)=P(B)P(B∣A)P(A)​
  • AI Terminology:
    • P(A∣B)P(A | B)P(A∣B): Posterior Probability (What we want to know; probability of hypothesis AAA given evidence BBB)
    • P(B∣A)P(B | A)P(B∣A): Likelihood (How likely is the evidence BBB given the hypothesis AAA)
    • P(A)P(A)P(A): Prior Probability (Initial belief in hypothesis AAA before seeing evidence)
    • P(B)P(B)P(B): Evidence / Marginal Likelihood (Probability of observing evidence BBB) Posterior=Likelihood×PriorEvidence\text{Posterior} = \frac{\text{Likelihood} \times \text{Prior}}{\text{Evidence}}Posterior=EvidenceLikelihood×Prior​

Applying Bayes' Theorem (Dentist Example)​

  • Goal: Find diagnostic rate P(cavity∣toothache)P(\text{cavity} | \text{toothache})P(cavity∣toothache).
  • Given (Medical Knowledge):
    • Likelihood: P(toothache∣cavity)=0.6P(\text{toothache} | \text{cavity}) = 0.6P(toothache∣cavity)=0.6
    • Prior: P(cavity)=0.2P(\text{cavity}) = 0.2P(cavity)=0.2
    • Evidence: P(toothache)=0.16P(\text{toothache}) = 0.16P(toothache)=0.16
  • Calculation: P(cavity∣toothache)=P(toothache∣cavity)P(cavity)P(toothache)=0.6×0.20.16=0.120.16=0.75P(\text{cavity} | \text{toothache}) = \frac{P(\text{toothache} | \text{cavity})P(\text{cavity})}{P(\text{toothache})} = \frac{0.6 \times 0.2}{0.16} = \frac{0.12}{0.16} = 0.75P(cavity∣toothache)=P(toothache)P(toothache∣cavity)P(cavity)​=0.160.6×0.2​=0.160.12​=0.75

Quick Summary​

  • Deterministic Logic Fails: Real-world uncertainty creates infinite exceptions, making pure logic intractable.
  • Probabilistic AI: Shifts from True/False to Degrees of Belief, acknowledging missing information.
  • Uncertainty Sources: Laziness (too many rules), Theoretical Ignorance (no perfect model), Practical Ignorance (limited sensors).
  • Random Variable (RV): Maps atomic events (ω\omegaω) to domain values (DDD).
  • Sample Space (Ω\OmegaΩ): Set of all mutually exclusive, exhaustive atomic events.
  • Kolmogorov's Axioms: Define valid probabilities (0≤P(ω)≤10 \le P(\omega) \le 10≤P(ω)≤1, ∑P(ω)=1\sum P(\omega) = 1∑P(ω)=1, Inclusion-Exclusion).
  • Prior Probability: Baseline belief before evidence.
  • Joint Probability Distribution (JPD): Table of all atomic event probabilities; powerful but fails catastrophically at scale (2n2^n2n entries).
  • Conditional Probability: P(A∣B)=P(A∧B)P(B)P(A | B) = \frac{P(A \land B)}{P(B)}P(A∣B)=P(B)P(A∧B)​, belief in A given B.
  • Product Rule: P(A∧B)=P(A∣B)P(B)P(A \land B) = P(A | B)P(B)P(A∧B)=P(A∣B)P(B), for building complex joint probabilities.
  • Bayes' Theorem: P(A∣B)=P(B∣A)P(A)P(B)P(A | B) = \frac{P(B | A)P(A)}{P(B)}P(A∣B)=P(B)P(B∣A)P(A)​, the AI engine for inverting probabilities and diagnostic reasoning (Posterior = Likelihood x Prior / Evidence).
