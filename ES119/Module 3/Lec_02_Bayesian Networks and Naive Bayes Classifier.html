Bayesian Networks and Naive Bayes Classifier

Recap: The Diagnostic Query​

We revisit Bayes' Rule with a Dentist system to calculate the probability of a cause given an effect.

Setup (Priors):

  • Prior probability of Cavity: P(c)=0.2P(c) = 0.2P(c)=0.2.
  • Prior probability of No Cavity: P(¬c)=0.8P(\neg c) = 0.8P(¬c)=0.8.

Sensor Model (Likelihoods):

  • Probe catches if Cavity: P(h∣c)=0.9P(h|c) = 0.9P(h∣c)=0.9.
  • Probe catches if No Cavity: P(h∣¬c)=0.2P(h|\neg c) = 0.2P(h∣¬c)=0.2.

Objective: Find P(c∣h)P(c|h)P(c∣h) (probability of cavity given probe catches).

The Denominator Trap: Marginalization​

To solve P(c∣h)P(c|h)P(c∣h), Bayes' Rule requires calculating P(h)P(h)P(h), the total evidence. P(c∣h)=P(h∣c)P(c)P(h)P(c|h) = \frac{P(h|c)P(c)}{P(h)}P(c∣h)=P(h)P(h∣c)P(c)​

The denominator P(h)P(h)P(h) is found by Marginalization: summing out all hidden variables. P(h)=∑C∈{c,¬c}P(h∧C)=P(h∧c)+P(h∧¬c)P(h) = \sum_{C \in \{c, \neg c\}} P(h \land C) = P(h \land c) + P(h \land \neg c)P(h)=∑C∈{c,¬c}​P(h∧C)=P(h∧c)+P(h∧¬c) P(h)=P(h∣c)P(c)+P(h∣¬c)P(¬c)P(h) = P(h|c)P(c) + P(h|\neg c)P(\neg c)P(h)=P(h∣c)P(c)+P(h∣¬c)P(¬c) For our example:

  • P(h∧c)=0.9×0.2=0.18P(h \land c) = 0.9 \times 0.2 = 0.18P(h∧c)=0.9×0.2=0.18
  • P(h∧¬c)=0.2×0.8=0.16P(h \land \neg c) = 0.2 \times 0.8 = 0.16P(h∧¬c)=0.2×0.8=0.16
  • Total Evidence P(h)=0.18+0.16=0.34P(h) = 0.18 + 0.16 = 0.34P(h)=0.18+0.16=0.34.

Complexity Crisis​

Calculating P(h)P(h)P(h) for a system with NNN binary hidden variables involves summing 2N2^N2N terms.

  • 1 Variable: 2 terms
  • 10 Variables: 1,024 terms
  • 30 Variables: 1.07 Billion terms This makes exact calculation for large-scale AI mathematically unsolvable.

Beating the Trap: Normalization (α\alphaα)​

The denominator P(h)P(h)P(h) is a constant for all hypotheses (P(c∣h)P(c|h)P(c∣h) and P(¬c∣h)P(\neg c|h)P(¬c∣h)). We define a Normalization Constant α=1/P(h)\alpha = 1/P(h)α=1/P(h). Then, we can write the posterior distribution as: P(C∣h)=α⋅P(h∧C)=α⋅P(h∣C)P(C)P(C|h) = \alpha \cdot P(h \land C) = \alpha \cdot P(h|C)P(C)P(C∣h)=α⋅P(h∧C)=α⋅P(h∣C)P(C) For our example: P(C∣h)=α(P(h∣c)P(c),P(h∣¬c)P(¬c))P(C|h) = \alpha (P(h|c)P(c), P(h|\neg c)P(\neg c))P(C∣h)=α(P(h∣c)P(c),P(h∣¬c)P(¬c)) P(C∣h)=α(0.18,0.16)P(C|h) = \alpha (0.18, 0.16)P(C∣h)=α(0.18,0.16) Since P(c∣h)+P(¬c∣h)=1P(c|h) + P(\neg c|h) = 1P(c∣h)+P(¬c∣h)=1, we have α(0.18+0.16)=1  ⟹  α(0.34)=1  ⟹  α=1/0.34≈2.94\alpha(0.18 + 0.16) = 1 \implies \alpha(0.34) = 1 \implies \alpha = 1/0.34 \approx 2.94α(0.18+0.16)=1⟹α(0.34)=1⟹α=1/0.34≈2.94. P(c∣h)=2.94×0.18≈0.529P(c|h) = 2.94 \times 0.18 \approx 0.529P(c∣h)=2.94×0.18≈0.529 P(¬c∣h)=2.94×0.16≈0.471P(\neg c|h) = 2.94 \times 0.16 \approx 0.471P(¬c∣h)=2.94×0.16≈0.471

Why α\alphaα is Enough for Decisions​

For decision-making, an AI agent often only needs to identify the most likely state. The normalization constant does not change the relative order of probabilities. If P(h∧A)>P(h∧B)P(h \land A) > P(h \land B)P(h∧A)>P(h∧B), then P(A∣h)>P(B∣h)P(A|h) > P(B|h)P(A∣h)>P(B∣h). This allows an engineering shortcut: calculate unnormalized terms, compare, and normalize only at the very end if exact probabilities are needed.

Defining Absolute Independence​

Purpose: Simplify systems by identifying variables that do not influence each other.

Mathematical Definition: Variables AAA and BBB are absolutely independent if: P(A∣B)=P(A)P(A|B) = P(A)P(A∣B)=P(A) or equivalently: P(A∧B)=P(A)P(B)P(A \land B) = P(A)P(B)P(A∧B)=P(A)P(B)

AI Interpretation: Learning the value of BBB provides zero new information about AAA. The probability of AAA remains the same.

Rarity: Absolute independence is rare in real-world AI systems where variables are often correlated (e.g., Toothache and Catch).

Defining Conditional Independence​

Most variables are correlated. Conditional independence provides a mechanism to break these correlations given a common cause.

Scenario: Flight Delay (A) and Crowded Lounge (B) appear dependent. A common cause like a Snowstorm (C) links them.

Mathematical Definition: Variables AAA and BBB are conditionally independent given CCC if their joint probability factors into separate parts once CCC is known: P(A,B∣C)=P(A∣C)⋅P(B∣C)P(A, B | C) = P(A | C) \cdot P(B | C)P(A,B∣C)=P(A∣C)⋅P(B∣C)

Equivalent Equality (Redundancy Property): If AAA and BBB are conditionally independent given CCC, then: P(A∣B,C)=P(A∣C)P(A | B, C) = P(A | C)P(A∣B,C)=P(A∣C) This means that if we already know CCC, learning BBB provides no additional information about AAA. BBB becomes redundant data given CCC.

Proof of Equivalence: Start with the general product rule: P(A,B∣C)=P(A∣B,C)⋅P(B∣C)P(A, B | C) = P(A | B, C) \cdot P(B | C)P(A,B∣C)=P(A∣B,C)⋅P(B∣C). Substitute the product definition for conditional independence: P(A∣C)⋅P(B∣C)=P(A∣B,C)⋅P(B∣C)P(A | C) \cdot P(B | C) = P(A | B, C) \cdot P(B | C)P(A∣C)⋅P(B∣C)=P(A∣B,C)⋅P(B∣C). Divide both sides by P(B∣C)P(B | C)P(B∣C) (assuming P(B∣C)>0P(B|C) > 0P(B∣C)>0): P(A∣C)=P(A∣B,C)P(A | C) = P(A | B, C)P(A∣C)=P(A∣B,C). Both definitions describe the same lack of direct connection between AAA and BBB when CCC is known.

Introduction to Bayesian Networks​

Bayesian Networks are graphical models that represent probabilistic relationships among variables.

Formal Definition: A Bayesian Network is a Directed Acyclic Graph (DAG) where:

  • Nodes represent random variables.
  • Arrows represent direct causal influence.
  • Missing Arrows explicitly represent assumptions of conditional independence.

Construction Principle: Conditional independence is used to remove unnecessary arrows, replacing direct links between correlated effects with a shared causal parent. (e.g., Delay ←\leftarrow← Snowstorm →\rightarrow→ Crowd).

Storing Knowledge in Conditional Probability Tables (CPTs)​

The quantitative knowledge in a Bayesian Network is stored in Conditional Probability Tables (CPTs).

CPT Definition: For every node XiX_iXi​, we store its probability distribution conditional on its direct parents: P(Xi∣Parents(Xi))P(X_i | Parents(X_i))P(Xi​∣Parents(Xi​))

Benefit: Due to conditional independence assumptions encoded in the graph, we only need to define a node's probability relative to its direct parents, drastically simplifying dependencies and storage requirements compared to a full Joint Probability Distribution.

Global Semantics: The Chain Rule​

A Bayesian Network represents the full Joint Probability Distribution (JPD) of all variables through factorization using the Chain Rule.

Product Formula: For variables X1,…,XnX_1, \dots, X_nX1​,…,Xn​: P(X1,…,Xn)=∏i=1nP(Xi∣Parents(Xi))P(X_1, \dots, X_n) = \prod_{i=1}^n P(X_i | Parents(X_i))P(X1​,…,Xn​)=∏i=1n​P(Xi​∣Parents(Xi​))

Dental Example: For Cavity (C), Toothache (T), and Catch (H), with C→TC \to TC→T and C→HC \to HC→H: P(C,T,H)=P(C)×P(T∣C)×P(H∣C)P(C, T, H) = P(C) \times P(T | C) \times P(H | C)P(C,T,H)=P(C)×P(T∣C)×P(H∣C)

Causal vs. Diagnostic Reasoning​

Causal Reasoning: Bayesian Networks are typically constructed based on causal relationships, storing P(Effect∣Cause)P(\text{Effect} | \text{Cause})P(Effect∣Cause) in CPTs (e.g., P(Toothache∣Cavity)P(\text{Toothache} | \text{Cavity})P(Toothache∣Cavity)). Predicting symptoms from a known disease is a straightforward lookup.

Diagnostic Reasoning: Real-world queries are often diagnostic; we observe symptoms (Effects) and want to infer the probability of the underlying Disease (Cause). This requires "reversing the arrows" using Bayes' Rule: moving from P(E∣C)P(E|C)P(E∣C) to P(C∣E)P(C|E)P(C∣E).

Example: Diagnostic Inference P(c∣t)P(c|t)P(c∣t) Given: P(c)=0.2P(c)=0.2P(c)=0.2, P(¬c)=0.8P(\neg c)=0.8P(¬c)=0.8, P(t∣c)=0.6P(t|c)=0.6P(t∣c)=0.6, P(t∣¬c)=0.1P(t|\neg c)=0.1P(t∣¬c)=0.1.

  1. Calculate Unnormalized Weights (Numerator terms):
    • W(c)=P(t∣c)P(c)=0.6×0.2=0.12W(c) = P(t|c)P(c) = 0.6 \times 0.2 = 0.12W(c)=P(t∣c)P(c)=0.6×0.2=0.12
    • W(¬c)=P(t∣¬c)P(¬c)=0.1×0.8=0.08W(\neg c) = P(t|\neg c)P(\neg c) = 0.1 \times 0.8 = 0.08W(¬c)=P(t∣¬c)P(¬c)=0.1×0.8=0.08
  2. Normalize: Sum of weights =0.12+0.08=0.20= 0.12 + 0.08 = 0.20=0.12+0.08=0.20.
    • P(c∣t)=W(c)W(c)+W(¬c)=0.120.20=0.60P(c|t) = \frac{W(c)}{W(c) + W(\neg c)} = \frac{0.12}{0.20} = 0.60P(c∣t)=W(c)+W(¬c)W(c)​=0.200.12​=0.60 Takeaway: Observing a toothache updates the prior belief of a cavity from 20% to 60%.

The Naive Bayes Classifier​

Bayesian Networks structured as a Cause leading to multiple Effects are used for Bayesian Classifiers. The goal is to predict the most likely cause given observed effects.

Problem: To find P(C∣E1,…,En)P(C | E_1, \dots, E_n)P(C∣E1​,…,En​), Bayes' Rule is: P(C∣E1,…,En)=P(E1,…,En∣C)P(C)P(E1,…,En)P(C | E_1, \dots, E_n) = \frac{P(E_1, \dots, E_n | C)P(C)}{P(E_1, \dots, E_n)}P(C∣E1​,…,En​)=P(E1​,…,En​)P(E1​,…,En​∣C)P(C)​ Using normalization: P(C∣E1,…,En)=αP(C)P(E1,…,En∣C)P(C | E_1, \dots, E_n) = \alpha P(C) P(E_1, \dots, E_n | C)P(C∣E1​,…,En​)=αP(C)P(E1​,…,En​∣C) The term P(E1,…,En∣C)P(E_1, \dots, E_n | C)P(E1​,…,En​∣C) still requires an exponentially large joint probability table (2n2^n2n entries).

The Naive Assumption (Conditional Independence)​

To overcome the computational/storage issue, the Naive Bayes Classifier assumes: Every Effect (EiE_iEi​) is conditionally independent of any other Effect (EjE_jEj​), given the Cause (CCC). Ei⊥Ej∣CE_i \perp E_j | CEi​⊥Ej​∣C This allows the joint probability of effects to factorize into a simple product: P(E1,…,En∣C)≈∏i=1nP(Ei∣C)P(E_1, \dots, E_n | C) \approx \prod_{i=1}^n P(E_i | C)P(E1​,…,En​∣C)≈∏i=1n​P(Ei​∣C) Impact: Complexity for calculating P(E1,…,En∣C)P(E_1, \dots, E_n | C)P(E1​,…,En​∣C) drops from exponential (2n2^n2n) to linear (nnn).

The Final Formula​

Combining Bayes' Rule with the Naive Assumption, the Naive Bayes Classifier formula is: P(C∣E1,…,En)=αP(C)∏i=1nP(Ei∣C)P(C | E_1, \dots, E_n) = \alpha P(C) \prod_{i=1}^n P(E_i | C)P(C∣E1​,…,En​)=αP(C)∏i=1n​P(Ei​∣C) Where α\alphaα is the normalization constant.

Key Characteristics:​

  • Robust: Works surprisingly well even if the independence assumptions are technically violated in practice.
  • Efficient: Requires minimal data to train, as only individual CPTs (P(Ei∣C)P(E_i|C)P(Ei​∣C)) and priors (P(C)P(C)P(C)) are needed.
  • Scalable: Easily handles a large number of evidence variables (e.g., hundreds of words in an email for spam classification).

Quick Summary​

  • Marginalization leads to exponential complexity in Bayes' Rule's denominator.
  • Normalization (α\alphaα) provides a computational shortcut for decision-making by focusing on proportional likelihoods.
  • Absolute Independence is rare; Conditional Independence (P(A,B∣C)=P(A∣C)P(B∣C)P(A,B|C)=P(A|C)P(B|C)P(A,B∣C)=P(A∣C)P(B∣C)) is key for simplifying dependencies.
  • Bayesian Networks are DAGs using nodes for variables, arrows for direct influence, and missing arrows for conditional independence.
  • CPTs (P(Xi∣Parents(Xi))P(X_i | Parents(X_i))P(Xi​∣Parents(Xi​))) store local knowledge efficiently.
  • The Chain Rule allows a BN to represent the full JPD: P(X1,…,Xn)=∏i=1nP(Xi∣Parents(Xi))P(X_1, \dots, X_n) = \prod_{i=1}^n P(X_i | Parents(X_i))P(X1​,…,Xn​)=∏i=1n​P(Xi​∣Parents(Xi​)).
  • Diagnostic Reasoning (Cause from Effect) uses Bayes' Rule to reverse causal arrows.
  • The Naive Bayes Classifier applies conditional independence of effects given a cause, leading to an efficient and scalable classification model: P(C∣E1,…,En)=αP(C)∏i=1nP(Ei∣C)P(C | E_1, \dots, E_n) = \alpha P(C) \prod_{i=1}^n P(E_i | C)P(C∣E1​,…,En​)=αP(C)∏i=1n​P(Ei​∣C).
