Passive Reinforcement Learning

From MDPs to Reinforcement Learning​

In standard Markov Decision Processes (MDPs), we assume the agent possesses the complete tuple (S,A,P,R)(S, A, P, R)(S,A,P,R). With the "physics" (PPP) and "motivation" (RRR) known, the optimal policy π∗\pi^*π∗ is calculated offline using Bellman equations.

The RL Reality: The agent is dropped into an unknown world.

  • P(s′∣s,a)P(s'|s, a)P(s′∣s,a) is hidden: The agent doesn't know transition probabilities (e.g., slip chances).
  • R(s,a,s′)R(s, a, s')R(s,a,s′) is hidden: The agent only discovers rewards by experiencing them.
  • Consequence: Utilities cannot be calculated on a "whiteboard"; the agent must learn from a sequence of interactions or Experience Tuples: (s,a,s′,r)(s, a, s', r)(s,a,s′,r).

Passive RL Paradigm​

In Passive Reinforcement Learning, the agent's policy π\piπ is fixed. The agent follows the rulebook π(s)\pi(s)π(s) without deviation.

  • Objective: Policy Evaluation. Estimate the expected utility Uπ(s)U^\pi(s)Uπ(s) for every state sss under the fixed policy.
  • Stochasticity: Even with a fixed policy, trajectories are noisy. The agent might intend to go North but slip East, creating different episodes each run.
  • Episode (Trial): A sequence of transitions from an initial state to a terminal state.

Method 1: Direct Utility Estimation (DUE)​

DUE treats policy evaluation as a supervised learning problem. If we want the expected value of a state, we observe it many times and average the results.

The Algorithm​

  1. Input Data (XXX): The state sss visited.
  2. Target Label (YYY): The observed Reward-to-Go (gtg_tgt​) from that state until the end of the episode.
  3. Calculation: For each visit to state sss, calculate: gt=rt+γrt+1+γ2rt+2+…g_t = r_t + \gamma r_{t+1} + \gamma^2 r_{t+2} + \dotsgt​=rt​+γrt+1​+γ2rt+2​+…
  4. Estimation: After NNN visits, the utility is the empirical average: Uπ(s)≈1N∑i=1NgiU^\pi(s) \approx \frac{1}{N} \sum_{i=1}^N g_iUπ(s)≈N1​∑i=1N​gi​

Evaluation​

  • Pros: Simple to implement, model-free (no need for PPP), and unbiased.
  • Cons (The Fatal Flaw): It treats states as independent "buckets." It ignores the Bellman Equation constraint: Uπ(s)=R(s)+γ∑s′P(s′∣s,π(s))Uπ(s′)U^\pi(s) = R(s) + \gamma \sum_{s'} P(s'|s, \pi(s))U^\pi(s')Uπ(s)=R(s)+γ∑s′​P(s′∣s,π(s))Uπ(s′). Because it misses state connectivity, it is highly data-inefficient.

Method 2: Adaptive Dynamic Programming (ADP)​

ADP is a model-based approach. Instead of ignoring the hidden PPP and RRR, the agent builds an internal model of them.

The Three-Step Process​

  1. Act: Observe transitions and rewards while following π\piπ.
  2. Model: Estimate the transition model P^\hat{P}P^ and reward function R^\hat{R}R^ via counting: P^(s′∣s,a)=N(s,a,s′)N(s,a)\hat{P}(s'|s, a) = \frac{N(s, a, s')}{N(s, a)}P^(s′∣s,a)=N(s,a)N(s,a,s′)​
  3. Solve: Plug P^\hat{P}P^ and R^\hat{R}R^ into the simplified Bellman Equation for a fixed policy: Uπ(s)=∑s′P^(s′∣s,π(s))[R^(s,π(s),s′)+γUπ(s′)]U^\pi(s) = \sum_{s'} \hat{P}(s'|s, \pi(s)) \left[ \hat{R}(s, \pi(s), s') + \gamma U^\pi(s') \right]Uπ(s)=∑s′​P^(s′∣s,π(s))[R^(s,π(s),s′)+γUπ(s′)]

Evaluation​

  • Pros: Maximum data efficiency. Discovered information (like a penalty) propagates backward instantly to all related states.
  • Cons: Computationally expensive. Solving NNN linear equations after every step is intractable for large state spaces (e.g., 102010^{20}1020 states in Backgammon).

Method 3: Temporal Difference (TD) Learning​

TD learning is the "Missing Link": it is model-free (like DUE) but respects state connectivity (like ADP).

The Concept of Bootstrapping​

Instead of waiting for an episode to end to calculate total rewards, TD updates the estimate of a state the moment it transitions to the next state. It updates a guess based on another guess.

The TD Update Rule​

Every time the agent takes a step (s,a,s′,r)(s, a, s', r)(s,a,s′,r), it applies: Uπ(s)←Uπ(s)+α[r+γUπ(s′)−Uπ(s)]⏟TD ErrorU^\pi(s) \leftarrow U^\pi(s) + \alpha \underbrace{\left[ r + \gamma U^\pi(s') - U^\pi(s) \right]}_{\text{TD Error}}Uπ(s)←Uπ(s)+αTD Error[r+γUπ(s′)−Uπ(s)]​​

  • TD Target: r+γUπ(s′)r + \gamma U^\pi(s')r+γUπ(s′). This is treated as the "local truth."
  • Learning Rate (α\alphaα): Determines the step size toward the new error.
  • Model-Free Nature: Notice P(s′∣s,a)P(s'|s, a)P(s′∣s,a) is missing. TD accounts for probabilities by experiencing transitions at their natural frequency.

Convergence and Conditions​

To guarantee convergence to the true utility despite noise, the learning rate α\alphaα must decrease over time according to the Robbins-Monro Conditions:

  1. ∑t=1∞α(t)=∞\sum_{t=1}^{\infty} \alpha(t) = \infty∑t=1∞​α(t)=∞: Ensures the agent can reach any target value.
  2. ∑t=1∞α2(t)<∞\sum_{t=1}^{\infty} \alpha^2(t) < \infty∑t=1∞​α2(t)<∞: Ensures the updates eventually settle and stop "bouncing." Example: α(t)=1/t\alpha(t) = 1/tα(t)=1/t satisfies these conditions.

Quick Summary​

  • Passive RL focuses on policy evaluation (UπU^\piUπ) rather than optimization.
  • DUE averages observed rewards-to-go but ignores state connections, making it data-inefficient.
  • ADP builds a model (P^,R^\hat{P}, \hat{R}P^,R^) and solves linear equations; it is data-efficient but computationally "explosive."
  • TD Learning uses bootstrapping to update estimates mid-episode. It serves as a lightweight approximation of ADP, respecting the Bellman constraint without requiring an explicit model.
  • Robbins-Monro conditions are necessary for TD convergence in stochastic environments.
