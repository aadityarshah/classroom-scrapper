Introduction to k-Nearest Neighbors (k-NN)

The kkk-Nearest Neighbors (kkk-NN) algorithm is a non-parametric, instance-based learning algorithm used for classification and regression. It's a lazy learner, meaning it does not construct a general internal model during training; instead, it memorizes the training dataset.

1. The k-NN Algorithm Steps​

Consider a dataset of nnn training points, {(x1,y1),(x2,y2),…,(xn,yn)}\{(x_1, y_1), (x_2, y_2), \dots, (x_n, y_n)\}{(x1​,y1​),(x2​,y2​),…,(xn​,yn​)}, where each xix_ixi​ is a ddd-dimensional feature vector and yiy_iyi​ is its corresponding label. For instance, in our example:

  • xix_ixi​: A feature vector, typically (xi1,xi2)(x_{i1}, x_{i2})(xi1​,xi2​) representing (sizei,avgRi)(\text{size}_i, \text{avgR}_i)(sizei​,avgRi​).
  • yiy_iyi​: A label, such as {J,M}\{J, M\}{J,M} (Jamun, Mango).

To classify a new test point t=(t1,t2,…,td)t = (t_1, t_2, \dots, t_d)t=(t1​,t2​,…,td​):

  1. Calculate Distances: Compute the distance between ttt and every training point xix_ixi​ in the dataset. A common choice is the Euclidean distance: di=∣∣xi−t∣∣2=∑j=1d(xij−tj)2d_i = ||x_i - t||_2 = \sqrt{\sum_{j=1}^{d} (x_{ij} - t_j)^2}di​=∣∣xi​−t∣∣2​=∑j=1d​(xij​−tj​)2​ For 2D data, this simplifies to: di=(xi1−t1)2+(xi2−t2)2d_i = \sqrt{(x_{i1} - t_1)^2 + (x_{i2} - t_2)^2}di​=(xi1​−t1​)2+(xi2​−t2​)2​

  2. Find kkk Smallest Distances: Identify the kkk training points that have the smallest distances to the test point ttt.

  3. Assign Label: Assign the label to the test point ttt based on the majority class among its kkk nearest neighbors. In regression tasks, this might involve averaging the values of the kkk neighbors.

2. General Complexity Analysis for k-NN​

Let's analyze the computational resources (time and space) required by the kkk-NN algorithm.

2.1 Data Representation​

Assume the training data XXX is an n×dn \times dn×d matrix, where nnn is the number of training instances and ddd is the dimensionality of each instance: X=(x1⊤x2⊤⋮xn⊤)X = \begin{pmatrix} x_1^\top \\ x_2^\top \\ \vdots \\ x_n^\top \end{pmatrix}X=​x1⊤​x2⊤​⋮xn⊤​​​ Each xix_ixi​ is a ddd-dimensional vector.

2.2 Time Complexity of Distance Calculation (Step 1)​

For each training point xix_ixi​, calculating the Euclidean distance to the test point ttt involves:

  • ddd subtractions.
  • ddd squarings.
  • d−1d-1d−1 additions.
  • 1 square root (often optimized away if only comparing distances). This is approximately O(d)\mathcal{O}(d)O(d) operations per training point. Since there are nnn training points, the total time for Step 1 is O(nd)\mathcal{O}(nd)O(nd).

2.3 Time Complexity of Finding kkk Smallest Distances (Step 2)​

Given a list of nnn distances d1,d2,…,dnd_1, d_2, \dots, d_nd1​,d2​,…,dn​, we need to find the kkk smallest ones.

2.3.1 Approach 1: Iterative Search​

One naive approach is to repeatedly search for the smallest element kkk times.

  • First search: Find the smallest element in nnn steps (O(n)\mathcal{O}(n)O(n)).
  • Second search: Find the next smallest in n−1n-1n−1 steps (after removing the first smallest) (O(n)\mathcal{O}(n)O(n)).
  • Repeat kkk times. The total time complexity for this approach is O(nk)\mathcal{O}(nk)O(nk).

2.3.2 Approach 2: Sorting​

Another approach is to sort all nnn distances and then pick the first kkk elements.

Insertion Sort:

  • Mechanism: Builds the final sorted array (or list) one item at a time. It iterates through the input elements and inserts each element into its correct position in the already sorted part of the array.
  • Complexity:
    • Best Case: O(n)\mathcal{O}(n)O(n) (when the array is already sorted).
    • Worst Case: O(n2)\mathcal{O}(n^2)O(n2) (when the array is sorted in reverse order).
    • Average Case: O(n2)\mathcal{O}(n^2)O(n2).
  • Space: O(1)\mathcal{O}(1)O(1) (sorts in place).

Merge Sort:

  • Mechanism: A divide-and-conquer algorithm. It recursively divides the array into halves until it has single-element arrays (which are sorted by definition), and then merges those halves back together in sorted order.
  • Complexity:
    • Best Case: O(nlog⁡n)\mathcal{O}(n \log n)O(nlogn).
    • Worst Case: O(nlog⁡n)\mathcal{O}(n \log n)O(nlogn).
    • Average Case: O(nlog⁡n)\mathcal{O}(n \log n)O(nlogn).
  • Space: O(n)\mathcal{O}(n)O(n) (due to temporary arrays used during merging).

2.4 Why Algorithmic Efficiency over Absolute Time?​

When comparing algorithms, we focus on the number of "steps" or operations rather than absolute execution time. This is because:

  • System Differences: Actual execution time can vary drastically based on hardware, operating system, programming language, compiler, and other concurrent processes.
  • Algorithmic Efficiency: The number of steps directly reflects the algorithm's inherent efficiency, independent of the underlying system.

While a slower algorithm might appear faster for small input sizes due to constant factors or better hardware, the algorithm with better asymptotic complexity will eventually outperform it as the input size nnn grows. This emphasizes the importance of understanding how algorithms scale.

2.5 Overall Time Complexity of k-NN​

Combining the steps:

  • Distance calculation: O(nd)\mathcal{O}(nd)O(nd).
  • Finding kkk smallest distances:
    • Using iterative search: O(nk)\mathcal{O}(nk)O(nk).
    • Using efficient sorting (e.g., Merge Sort): O(nlog⁡n)\mathcal{O}(n \log n)O(nlogn).
    • Using a min-heap or selection algorithm (e.g., Quickselect) can find the kkk-th smallest element and thus the kkk smallest elements in O(n)\mathcal{O}(n)O(n) time on average. If kkk elements are extracted from a min-heap, it's O(n+klog⁡k)\mathcal{O}(n + k \log k)O(n+klogk).

So, the total time complexity for kkk-NN depends on how Step 2 is implemented:

  1. Iterative Search for kkk smallest: O(nd+nk+k)≡O(nd+nk)\mathcal{O}(nd + nk + k) \equiv \mathcal{O}(nd + nk)O(nd+nk+k)≡O(nd+nk).
  2. Sorting all distances (e.g., Merge Sort): O(nd+nlog⁡n+k)≡O(nd+nlog⁡n)\mathcal{O}(nd + n \log n + k) \equiv \mathcal{O}(nd + n \log n)O(nd+nlogn+k)≡O(nd+nlogn).
  3. Using Quickselect/Min-Heap for kkk smallest: O(nd+n)\mathcal{O}(nd + n)O(nd+n). (assuming klog⁡kk \log kklogk is dominated by nnn or kkk is small).

Typically, k≪nk \ll nk≪n. If kkk is small constant, nknknk becomes nnn, so O(nd+n)\mathcal{O}(nd + n)O(nd+n). If ddd is also small constant, the dominant term becomes nnn.

2.6 Space Complexity of k-NN​

The kkk-NN algorithm is instance-based, meaning it needs to store all training data to make predictions.

  • Storing the n×dn \times dn×d training matrix XXX: O(nd)\mathcal{O}(nd)O(nd).
  • Storing distances: O(n)\mathcal{O}(n)O(n). The dominant space complexity is O(nd)\mathcal{O}(nd)O(nd).

3. Asymptotic Notation: Big-O​

Asymptotic notation provides a way to describe the limiting behavior of functions, particularly the growth rate of an algorithm's running time or space requirements as the input size grows.

3.1 Definition of Big-O Notation​

A function f(n)f(n)f(n) is said to be O(g(n))\mathcal{O}(g(n))O(g(n)) (read as "Big-O of g(n)g(n)g(n)") if there exist positive constants ccc and n0n_0n0​ such that: f(n)≤c⋅g(n)for all n≥n0f(n) \le c \cdot g(n) \quad \text{for all } n \ge n_0f(n)≤c⋅g(n)for all n≥n0​ This means that for sufficiently large input sizes (n≥n0n \ge n_0n≥n0​), the growth rate of f(n)f(n)f(n) is bounded above by some constant multiple of g(n)g(n)g(n). In other words, g(n)g(n)g(n) is an asymptotic upper bound for f(n)f(n)f(n). It implies that g(n)g(n)g(n) is "not more efficient" (or potentially less efficient or equally efficient) than f(n)f(n)f(n) in terms of growth rate.

4. Practical Considerations and Enhancements for k-NN​

4.1 Choosing kkk​

The choice of kkk is crucial and impacts the algorithm's performance:

  • Small kkk: Makes the model sensitive to noise and outliers (low bias, high variance). It captures local structure well but can overfit.
  • Large kkk: Smooths out the decision boundary, reducing the impact of noise but potentially overlooking fine-grained local patterns (high bias, low variance). It can lead to underfitting.
  • Class Imbalance: In imbalanced datasets, a large kkk might lead to the majority class dominating predictions, even for points closer to the minority class.
  • Local Structure: The optimal kkk often reflects the underlying density and distribution of data points.

4.2 Other Drawbacks​

  • Feature Scaling: Different features (e.g., 'size' vs. 'avgR') often have different scales. Distance metrics are sensitive to these scales, meaning features with larger ranges can disproportionately influence the distance calculation. Normalization or standardization of features is critical.
  • High Test Time and Storage Requirements: As seen in the complexity analysis, kkk-NN requires significant computational time (O(nd)\mathcal{O}(nd)O(nd) or O(nd+nlog⁡n)\mathcal{O}(nd + n \log n)O(nd+nlogn)) and storage (O(nd)\mathcal{O}(nd)O(nd)) for large nnn and ddd. This makes it computationally expensive for large datasets, particularly during inference, and susceptible to the "Curse of Dimensionality."

4.3 Adaptive k-NN​

Instead of using a fixed kkk for all test points, adaptive kkk-NN chooses a specific k(x)k(x)k(x) for each new test point xxx.

  • One approach is to select k(x)k(x)k(x) based on the number of points within a certain ϵ\epsilonϵ-radius around xxx.
  • Another approach could be to choose k(x)k(x)k(x) to maximize a "confidence" score for the prediction. This allows for local adaptation of the neighborhood size.

4.4 Condensed k-NN​

Condensed kkk-NN aims to reduce the memory footprint and potentially the inference time by finding a smaller, representative subset of the training data. The general idea:

  1. Start with an empty condensed set CCC.
  2. Iterate through the original training set TTT.
  3. For each point xxx in TTT: a. Apply the kkk-NN algorithm using only the points currently in CCC to predict xxx's label. b. If the prediction for xxx is incorrect, add xxx to CCC.
  4. Repeat until no more points are added to CCC in an iteration, or a stopping criterion is met.
  5. Finally, during test time, use only the condensed set CCC for predictions.

This method helps prune redundant training instances, making the model more efficient while striving to maintain predictive performance.

Quick Summary​

This lecture covered the k-Nearest Neighbors (k-NN) algorithm, detailing its step-by-step process for classification based on distance metrics and majority voting. We then performed a thorough complexity analysis, distinguishing between the O(nd)\mathcal{O}(nd)O(nd) cost for distance calculations and various costs for finding the kkk nearest neighbors (O(nk)\mathcal{O}(nk)O(nk) for iterative search, O(nlog⁡n)\mathcal{O}(n \log n)O(nlogn) for sorting-based methods). The lecture emphasized the critical role of Big-O notation in evaluating algorithmic scalability by providing an asymptotic upper bound on growth rates, highlighting why it's preferred over absolute time measurements. Finally, we discussed practical challenges of k-NN, such as feature scaling and high computational demands, and introduced adaptive k-NN and condensed k-NN as strategies to address these limitations.
